JavaScript Reference VS Copy

Strings, numbers, and booleans are basic data types. Here's a brief explanation:

Strings

        
        
        const originalString = 'Hello, world!';
        const copiedString = originalString.slice();
        console.log(originalString, copiedString); // 'Hello, world!' 'Hello, world!'

Strings are copied using methods like slice(). Modifying the copied string doesn't affect the original string.

Numbers

      
      
      const originalNumber = 42;
      const copiedNumber = originalNumber;
      console.log(originalNumber, copiedNumber); // 42 42

Numbers are copied by direct assignment (=). Modifying the copied number affects the original number.

Boolean

      
      
      const originalBoolean = true;
      const copiedBoolean = originalBoolean;
      console.log(originalBoolean, copiedBoolean); // true true

Booleans are copied by direct assignment (=). Modifying the copied boolean affects the original boolean.

Copying Arrays

      
        
       // Let's say we have an array
        const players1 = ['Wes', 'Sarah', 'Ryan', 'Poppy'];

        // Method 1: Using slice() to create a copy (reference)
        const playersCopy1 = players1.slice();
        playersCopy1.push('NewPlayer1');
        console.log(players1, playersCopy1);

        // Method 2: Using concat() to create a copy (reference)
        const players2 = ['Wes', 'Sarah', 'Ryan', 'Poppy'];
        const playersCopy2 = [].concat(players2);
        playersCopy2.push('NewPlayer2');
        console.log(players2, playersCopy2);

        // Method 3: Using the spread operator (...) to create a copy (reference)
        const players3 = ['Wes', 'Sarah', 'Ryan', 'Poppy'];
        const playersCopy3 = [...players3];
        playersCopy3.push('NewPlayer3');
        console.log(players3, playersCopy3);

Each method creates a reference to the original array, not an independent copy. Therefore, modifying the copy also affects the original array.

Conclusion: It's important to be aware that these methods result in shallow copies, meaning they only go one level deep. If the array contains nested objects, modifying the nested objects within the copy will affect the original array. For a deep copy, consider using external libraries like lodash's cloneDeep.

Copying Objects

      
        // With Objects
        const person = {
          name: 'Wes Bos',
          age: 80
        };

        // Method 1: Using Object.assign() to create a copy (reference)
        const personCopy1 = Object.assign({}, person);
        personCopy1.age = 30;
        console.log(person, personCopy1);

        // Method 2: Using the spread operator (...) to create a copy (reference)
        const personCopy2 = { ...person };
        personCopy2.age = 25;
        console.log(person, personCopy2);

Objects can be copied using methods like Object.assign() or the spread operator (...).
Method 1 (Object.assign()): Creates a new object by copying properties from the original object.
Method 2 (Spread Operator): Creates a new object with the same properties as the original object. Modifying the copied object doesn't affect the original object.
